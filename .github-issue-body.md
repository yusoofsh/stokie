### Description

The admin client plugin (`adminClient`) only exposes the `checkRolePermission` method in its TypeScript type definitions. All other admin methods such as `listUsers`, `createUser`, `setRole`, `banUser`, `unbanUser`, `revokeUserSessions`, `impersonateUser`, and `stopImpersonating` are not available on the `auth.admin` object according to TypeScript, even though they should be available based on the plugin's functionality.

### Current Status

**Impact:** Blocking - TypeScript compilation fails when trying to use any admin methods other than `checkRolePermission`.

### Expected Status

All admin plugin methods should be properly typed and accessible on the `auth.admin` object when using the `adminClient` plugin, including:
- `listUsers`
- `createUser`
- `setRole`
- `banUser`
- `unbanUser`
- `revokeUserSessions`
- `impersonateUser`
- `stopImpersonating`

### Environment

- **better-auth version:** 1.4.9
- **Plugin configuration:**
  - Server: `admin()` plugin enabled in `betterAuth` config
  - Client: `adminClient()` plugin enabled in `createAuthClient` config

### Code Example

**Client setup (src/lib/auth/client.ts):**
```typescript
import { adminClient } from "better-auth/client/plugins"
import { createAuthClient } from "better-auth/react"

export const auth = createAuthClient({
  plugins: [adminClient()],
})
```

**Usage attempt (src/routes/dashboard/admin/users.tsx:27):**
```typescript
const data = await auth.admin.listUsers({
  query: {
    limit: 10,
    sortBy: "createdAt",
    sortDirection: "desc",
  },
})
```

TypeScript error: `Property 'listUsers' does not exist on type '{ checkRolePermission: ...`

### Possible Solutions

**Verified Option:**
- The admin plugin server endpoints exist and are properly defined in `packages/better-auth/src/plugins/admin/routes.ts`
- The client plugin type inference may not be correctly mapping the server plugin's endpoints to the client API

**Speculative Option:**
- The `$InferServerPlugin` mechanism in `adminClient` may not be properly inferring all endpoint methods from the server `admin` plugin
- Type generation or path-to-object mapping for admin routes may need adjustment
